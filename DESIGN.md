# DoodleIP — 不露脸，也能让别人记住你

给内容创作者的专属 IP 生成器

更新日期：2026-04-03
状态：已批准

## 一句话定位

**"不露脸，也能让别人记住你"** — 给内容创作者的专属 IP 生成器

## 核心洞察

**把你的内容，变成你的 IP。**

很多人想做内容但不想露脸，或者已经在做文字内容但没有视觉辨识度。DoodleIP 给了一条路：一个专属角色替你"出镜"，让你的内容从此有脸、有辨识度、被人记住。

## 用户是谁

小红书/Instagram 上的内容创作者，两类混合体：

- **已经在发内容的：** 干货/观点/教程博主，全是纯文字截图，在信息流里没有辨识度
- **想开始但没迈出第一步的：** 有内容能力（能写），但不想露脸，不会设计，不知道怎么建立视觉风格

**共同特征：** 有内容，缺视觉身份。不露脸，但想被记住。

## 解决什么问题

纯文字内容在小红书信息流里是隐形的。用户刷过就忘了你是谁。

像机器坏人那种形式 — 一个简笔 IP 角色贯穿所有帖子 — 让人一眼认出"又是这个人"，辨识度和粘性完全不同。但普通人做不出来：不会画、不会设计、没有工具能帮你从零拥有一个 IP。

## 价值主张

**不是模板工具，不是效率工具。是身份工具。**

帮你从"无脸博主"变成"有 IP 的创作者"。你的内容还是你的内容，但从此有脸了。

**竞争对手不是 Canva，是"继续发没有辨识度的纯文字"这个现状。**

## 产品体验

### IP 创建（Onboarding — 同时也是核心时刻）

IP 是永久的，跟着你一直用。创建 IP 既是 onboarding，也是最激动人心的时刻。

1. 用户描述自己想要的角色："一只戴眼镜的猫"或"一个圆脑袋火柴人围着红围巾"
2. LLM 生成详细的风格 prompt
3. FLUX Kontext / GPT Image 1.5 生成 3-4 个候选角色
4. 用户选择一个
5. 系统以选中的角色为参考，生成 6-8 个核心姿势变体
6. 姿势存储在用户的素材库中（Cloudflare R2）
7. 如果某个姿势质量不合格（无法辨认），用更严格的 prompt 重新生成，或回退到默认姿势库

### 内容生成（日常循环）

1. 用户输入文字（文章、笔记、要点列表）
2. LLM 分析并拆分为 5-7 页轮播图
3. 对每一页：选择模板 + 选择角色姿势 + 注入文字
4. 模板引擎渲染所有页面
5. 用户预览并可调整：替换姿势、编辑文字、切换模板、重新排序
6. 下载 ZIP，直接发小红书/Instagram

### 核心时刻

用户第一次看到"自己的 IP 角色出现在一组可以直接发的轮播图里"。

## 技术方案（方案 B：模板渲染 + AI 角色）

### 架构

```
用户输入（文字）
    |
    v
[LLM 层] Claude/GPT
    - 将文字拆分为 5-7 页
    - 决定页面类型：封面 / 正文 / 总结
    - 为每页选择角色姿势
    - 选择高亮颜色、强调样式
    |
    v
[角色素材库]
    - 在 onboarding 阶段通过 FLUX Kontext / GPT Image 1.5 生成
    - MVP：6-8 个核心姿势（站立、坐着、思考、指向、挥手、拿手机、打字、庆祝）
    - 透明 PNG，风格统一
    - 按用户存储在 Cloudflare R2（便宜、快、S3 兼容）
    |
    v
[模板引擎] HTML/Canvas → PNG
    - 每种页面类型有多个布局模板
    - 注入：文字内容 + 角色姿势 + 品牌色
    - 渲染为 1080x1350px（Instagram/小红书标准）
    |
    v
[输出]
    - 5-7 张轮播图
    - 下载为 ZIP
```

### 技术栈

- Next.js + Vercel（免费层，自动部署）
- Stripe Checkout（最简单的支付）
- HTML + Puppeteer 模板渲染（中文字体支持好）
- FLUX Kontext via fal.ai 或 GPT Image 1.5 via OpenAI API（角色生成）
- Cloudflare R2（素材存储）

## MVP 范围

**做：**
- 角色创建（描述 → 候选 → 选择 → 姿势生成）
- 文字拆分为轮播页面（LLM）
- 模板渲染（3 类页面 x 2 布局 = 6 模板）
- 下载 ZIP
- Stripe Checkout 订阅

**不做：**
- 账号系统（v1 先不做）
- 角色管理/历史
- 风格选择器
- 直接发布到平台
- 内容日历

## 失败模式

| 失败场景 | 影响 | 应对策略 |
|---------|------|---------|
| API 生成不一致的姿势 | 角色无法辨认 | 用更严格的 prompt 重新生成；回退到默认姿势库 |
| API 宕机/限流 | Onboarding 阻塞 | 排队+重试；显示"生成中..."状态；积极缓存 |
| LLM 内容拆分不合理 | 轮播图节奏差 | 用户可以在渲染前手动编辑页面拆分 |
| 模板放不下内容 | 文字溢出/太空 | LLM 遵守字数限制；模板自动调整字号 |
